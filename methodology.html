Methodology

The website applies decision support theory to recommend the most suitable smartphone.

This system is based on Decision Support System theory, multi-criteria evaluation, and weighted utility scoring. User preferences are converted into measurable weights, then each phone alternative is evaluated and ranked.

Core Formula Score = (Price x Wp) + (Camera x Wc) + (Battery x Wb) + (Performance x Wf)

Each criterion contributes to the total utility score according to the user-defined importance weight.

Research Goal

To help users choose the best smartphone by comparing alternatives across several criteria rather than relying only on brand preference or price.

Decision Support System

A Decision Support System is an information system that supports users in making semi-structured decisions. In this project, the DSS assists users by organizing phone data, applying evaluation rules, and producing recommendation results.

Multi-Criteria Decision Making

Smartphone selection is a multi-criteria problem because users consider more than one factor at the same time. Price, camera, battery, and performance must be evaluated together.

Utility-Based Ranking

Utility theory is used to convert different phone attributes into one comparable value. The model with the highest final utility score becomes the recommended alternative.

Theoretical Foundation

1. Decision Theory

Decision theory explains how a person can choose the best option among many alternatives. In this website, each smartphone is treated as an alternative and each phone feature is treated as an evaluation criterion.

A good decision model should reduce uncertainty, organize information clearly, and provide a rational basis for choosing one option over another. The DSS performs this role by transforming user needs into a ranked recommendation.

Criteria Theory

2. Evaluation Criteria

Price

Represents the economic cost of the phone. It is an important factor because many users make decisions within a limited budget.

Camera

Reflects image and video capability. This criterion is important for users who prioritize photography, content creation, or social media usage.

Battery

Measures power endurance and practical daily usage. A higher battery score means better long-term use without frequent charging.

Performance

Represents processing speed, responsiveness, and ability to handle demanding tasks such as gaming, multitasking, and productivity work.

Scoring Model

3. Weighted Sum Model

The website uses the Weighted Sum Model because it is simple, explainable, and suitable for ranking alternatives across several criteria. Each phone receives a final score based on the sum of its weighted criterion values.

The formula used in the system is:

Score = (Price x Price Weight) + (Camera x Camera Weight) + (Battery x Battery Weight) + (Performance x Performance Weight)

After calculating the total score for every selected model, the phones are sorted in descending order. The highest score is interpreted as the best recommendation.

Preference Theory

4. User-Defined Weights

Different users have different priorities. Some users prefer a lower price, while others want the best camera or strongest performance. Therefore, the model allows the user to assign weights to each criterion.

Weights represent importance. A higher weight means the user values that criterion more. This makes the recommendation personalized rather than fixed for everyone.

Data Handling

5. Dataset and Attribute Representation

The system dataset contains iPhone and Samsung smartphone models from 2010 to 2025. Each record contains the model name, brand, year, price, camera score, battery score, and performance score.

These attributes are used as structured decision variables. Organizing the data in JSON format makes the system easy to update, process, and integrate with the web interface and Flask API.

Process Flow

6. System Procedure

Step 1. The user views the available smartphone dataset.

Step 2. The user selects or considers one or more smartphone models.

Step 3. The user enters preference weights for price, camera, battery, and performance.

Step 4. The system calculates a utility score for each model.

Step 5. The system ranks all alternatives from highest to lowest.

Step 6. The system presents the winner and recommendation summary.

Interpretation

7. Ranking Output

The ranking list helps users compare all evaluated phones, not only the winner. This improves transparency because the user can see how close or far each alternative is from the top recommendation.

Normalization Idea

8. Comparative Measurement

In decision analysis, criteria often come from different value ranges. Even when a simple scoring model is used, the theoretical purpose is to bring different attributes into a comparable decision space so they can contribute to one overall result.

Recommendation Logic

9. Why the Winner Is Chosen

The recommended phone is the alternative with the highest total weighted score. This means it offers the strongest overall balance according to the user's own priorities.

Academic Value

10. Research Contribution

This project demonstrates how DSS concepts can be applied to a real consumer problem. It shows that structured evaluation can improve smartphone purchasing decisions by making the recommendation process more objective, transparent, and personalized.

Input

Phone dataset and user-defined weights are entered into the system as decision inputs.

Process

Weighted scoring, ranking, and comparison are applied to evaluate all selected alternatives.

Output

The final output includes utility scores, ranking order, winner result, and recommendation advice.
